# peer_service

client_service와 함께 분산 처리를 완결한다. 

관계가 많이 생긴다. 
- 로컬 액터와 리모트 액터의 up / down 사용 인터페이스
- 피어의 down 시 리모트 액터 down 처리 

## use case 

리모트 액터를 쉽게 사용하도록 하는 것이 핵심 기능이다. 
리모트 액터가 로컬로 옮겨오더라도 동일한 코드로 동작해야 한다. 
이것이 관건이다. 

## 단계적 구현 

- 구성 파일 읽기 
- listen 
- connect
- 핸들러 등록 
- 핸들러 구현 
- 필요한 데이터 추가 
- syn_peer_up / syn_peer_down 메세지 추가
